## Задача

Предсказывать `Win / Draw / Loss` по признакам:

- Разница в рейтинге (сильный сигнал) `MyRate - OppRate`
- Номер хода (обстоятельство) `__Moves[ply]` 
- Оставшееся время (главный интерес) `__Moves[clock_after]`


## Интерпретация модели

Чтобы при средней разнице в рейтингах `rating_diff = 0`  
вероятность выигрыша `P(win)` была не менее, чем  `45%` в среднем,  
к ходу номер `move_N = 15` я должен уложиться во время `clock > ?`

- Модель дает `P(win | clock, move_N, rating_diff)` 
- Фиксируем `move_N = 15`, `rating_diff = 0`
- Строим кривую `P(win)` по `clock`
- Находим граничное значение `clock = ?`


## Модели

- Baseline: **Logistic Regression** как это часто бывает.  
Простая, легко интерпретируемая.

- Основная: **Любой Boost**.  
Нелинейности, и взаимодействия признаков.

Недавние партии отражают мой текущий уровень лучше, чем старые.  
Предполагаю, что вклад давности игры `Date` в  
предсказание – это экспоненциальное затухание `exp(-λ * days_ago)`.

Time-based split.
